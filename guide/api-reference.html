üîç Core Functions

extractAllElements()

contentScript.js Core Function

Main function that scans the entire DOM and extracts all relevant elements.

function extractAllElements(): Array<ElementData>

Returns

  • Array<ElementData> - Array of analyzed element objects

Description

Traverses the DOM tree, identifies interactive and significant elements, analyzes each element for metadata and selectors, and returns a structured array of element data.

Example

// Extract all elements from current page
const elements = extractAllElements();
console.log(`Found ${elements.length} elements`);

// Process results
elements.forEach(element => {
    console.log(`${element.tagName}: ${element.selector}`);
});

shouldExtractElement(element)

contentScript.js Filter Function

Determines whether an element should be included in extraction results.

function shouldExtractElement(element: HTMLElement): boolean

Parameters

  • element (HTMLElement) - DOM element to evaluate

Returns

  • boolean - True if element should be extracted

Filter Criteria

  • Element is visible (has dimensions)
  • Element is interactive or has content
  • Element is not a script or style tag
  • Element meets minimum size requirements

Example

const button = document.querySelector('#submit-btn');
if (shouldExtractElement(button)) {
    console.log('Button will be extracted');
}

analyzeElement(element)

contentScript.js Analysis Function

Performs comprehensive analysis of a DOM element and returns structured data.

function analyzeElement(element: HTMLElement): ElementData

Parameters

  • element (HTMLElement) - DOM element to analyze

Returns

  • ElementData - Complete element analysis object

Analysis Includes

  • Basic properties (tag, id, classes)
  • Content extraction (text, value, attributes)
  • Position and dimensions
  • CSS and XPath selectors
  • Visibility and interaction state

Example

const input = document.querySelector('#email');
const analysis = analyzeElement(input);

console.log(analysis);
// Output: {
//   tagName: 'INPUT',
//   id: 'email',
//   selector: '#email',
//   textContent: '',
//   value: 'user@example.com',
//   ...
// }

üéØ Selector Generation Functions

generateCSSSelector(element)

contentScript.js Selector Function

Generates the most specific and reliable CSS selector for an element.

function generateCSSSelector(element: HTMLElement): string

Selector Priority

  1. ID selector: #unique-id
  2. Data attributes: [data-testid="value"]
  3. Class selector: .class-name
  4. Attribute selector: [name="field"]
  5. Tag with attributes: input[type="email"]
  6. Hierarchical path: .parent > .child

Example

const input = document.querySelector('#email-field');
const selector = generateCSSSelector(input);
console.log(selector); // "#email-field"

const button = document.querySelector('.btn.primary');
const selector2 = generateCSSSelector(button);
console.log(selector2); // ".btn.primary"

generateXPathSelector(element)

contentScript.js Selector Function

Creates XPath expressions for robust element location.

function generateXPathSelector(element: HTMLElement): string

XPath Strategies

  • Attribute-based: //button[@id='submit']
  • Text-based: //button[text()='Click Me']
  • Position-based: //div[1]/button[2]
  • Class-based: //div[@class='container']

Example

const submitBtn = document.querySelector('#submit');
const xpath = generateXPathSelector(submitBtn);
console.log(xpath); // "//button[@id='submit']"

const textLink = document.querySelector('a');
const xpath2 = generateXPathSelector(textLink);
console.log(xpath2); // "//a[text()='Learn More']"

getOptimalSelector(element)

contentScript.js Smart Selector Function

Returns the best selector strategy for a given element based on uniqueness and stability.

function getOptimalSelector(element: HTMLElement): {
  css: string,
  xpath: string,
  recommended: string,
  strategy: string
}

Selection Logic

  • Tests multiple selector strategies
  • Validates uniqueness of each selector
  • Considers stability and maintenance
  • Returns recommendation with reasoning

Example

const element = document.querySelector('.complex-element');
const selectors = getOptimalSelector(element);

console.log(selectors);
// {
//   css: ".form-container .submit-btn",
//   xpath: "//div[@class='form-container']//button",
//   recommended: "#submit-button",
//   strategy: "id-based"
// }

displayResults(elements)

popup.js UI Function

Renders extracted elements in the popup table with pagination.

function displayResults(elements: Array<ElementData>): void

Features

  • Paginated table display (15 items per page)
  • Sortable columns
  • Copy individual selectors
  • Real-time search/filter
  • Element highlighting on click

Example

chrome.runtime.onMessage.addListener((message) => {
  if (message.action === 'elementsExtracted') {
    displayResults(message.elements);
    updateStats(message.elements.length);
  }
});

üìã Data Structures

ElementData

Interface

Primary data structure representing an analyzed DOM element.

interface ElementData {
  tagName: string;
  id: string;
  classes: string[];
  textContent: string;
  value: string;
  selector: string;
  xpath: string;
  attributes: Record<string, string>;
  position: Position;
  dimensions: Dimensions;
  isVisible: boolean;
  isInteractive: boolean;
  elementType: ElementType;
}

Properties

  • tagName: HTML tag name (uppercase)
  • id: Element ID attribute
  • classes: Array of CSS class names
  • textContent: Visible text content
  • value: Form element value
  • selector: Optimal CSS selector
  • xpath: XPath expression
  • attributes: Element attributes object
  • position: X, Y coordinates
  • dimensions: Width, height
  • isVisible: Visibility state
  • isInteractive: Interactivity state
  • elementType: Classification

Position

Interface

Element position coordinates.

interface Position {
  x: number;
  y: number;
}

Dimensions

Interface

Element size dimensions.

interface Dimensions {
  width: number;
  height: number;
}

ElementType

Enum

Element classification categories.

enum ElementType {

  LINK = 'link',
  IMAGE = 'image',
  TEXT = 'text',
  CONTAINER = 'container',
  INTERACTIVE = 'interactive',
  OTHER = 'other'

üì° Event System

Chrome Messaging API

Communication

Message passing between popup and content script.

Message Types

// Popup to Content Script
{
  action: 'extractElements',
  timestamp: Date.now()
}

// Content Script to Popup
{
  action: 'elementsExtracted',
  elements: ElementData[],
  count: number,
  timestamp: Date.now()
}

// Error messages
{
  action: 'error',
  message: string,
  details: any
}

Usage Example

// Send message from popup
chrome.tabs.sendMessage(tabId, {
  action: 'extractElements'
});

// Listen in content script
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  if (message.action === 'extractElements') {
    const elements = extractAllElements();
    sendResponse({
      action: 'elementsExtracted',
      elements: elements,
      count: elements.length
    });
  }
});

üö® Error Handling

Common Error Types

  • Permission Errors: Script injection blocked
  • Cross-Origin Errors: Iframe access denied
  • DOM Errors: Element not found or invalid
  • Message Errors: Communication timeout
  • Export Errors: File generation failed

Error Handling Patterns

try {
  const elements = extractAllElements();
  chrome.runtime.sendMessage({
    action: 'elementsExtracted',
    elements: elements
  });
} catch (error) {
  chrome.runtime.sendMessage({
    action: 'error',
    message: error.message,
    details: error.stack
  });
}
